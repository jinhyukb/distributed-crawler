# 📚 분산형 실시간 데이터 수집 및 가용성 관제 시스템 (Distributed Crawler)

이 프로젝트는 대용량 분산 환경에 바로 적용할 수 있는 수준의 실무형 파이썬 크롤링 파이프라인 시스템입니다. 단일 크롤러 엔진의 한계를 극복하기 위해 분산 태스크 분배, 중복 수집 예외 처리, 데이터 정합성 검증, 데이터베이스 트랜잭션, 그리고 시스템 가용성 실시간 모니터링 체계를 완전히 구현했습니다.

---

## 🚀 주요 핵심 기술 특징

1. **분산 태스크 제어 및 중복 제거 (Redis)**
   - 여러 워커 노드가 병렬로 수집 작업을 수행해도 충돌 없이 순차 처리하는 FIFO 분산 큐 아키텍처를 도입했습니다.
   - 이미 수집이 종료된 주소는 중복 필터(`crawler:task:visited`)를 통해 데이터베이스 및 대기열 쓰기 단계 이전에 원천 차단합니다.
2. **엄격한 런타임 데이터 정제 (Pydantic Schema)**
   - 수집된 원본 HTML에서 불규칙한 화폐 기호 정제나 통계 정량화를 수집 단계에서 즉시 수행합니다.
   - 필수 필드 누락이나 데이터 오염 발생 시, 데이터베이스 저장소로 오염된 데이터가 인입되지 않도록 격리 및 추적합니다.
3. **영속성 데이터 관리 (RDBMS UPSERT)**
   - SQL 표준 규격을 준수하는 SQLite DB 엔진을 사용합니다.
   - 고속 벌크 트랜잭션(`executemany`) 기법을 통해 디스크 I/O 병목을 줄였습니다.
   - 수집 중 중복 데이터 발생 시, 충돌 에러로 전체 시스템이 중단되는 대신 기존 정보를 안전하게 신규 스키마로 갱신(`UPSERT`)합니다.
4. **시스템 관제 및 임계치 경고 엔진**
   - 크롤러의 가동률, 지연 시간(Latency), 에러율을 실시간으로 수합하는 메트릭 수집기가 탑재되어 있습니다.
   - 타겟 웹페이지의 개편으로 구조적 파싱 에러가 다량 발생하거나 IP 차단 위기 발생 시, 알림 트리거를 발송합니다.
5. **독립 가상 컨테이너 환경 제공 (Docker)**
   - `Dockerfile` 및 `docker-compose.yml`을 제공하여 어떠한 환경에서도 라이브러리 충돌이나 가상 인프라(Redis) 설정 번거로움 없이 하나의 명령어로 구동할 수 있습니다.

## 📂 디렉터리 구조 및 파일 설명

```text
distributed-crawler/
├── .github/
│   └── workflows/
│       └── deploy.yml          # GitHub Actions를 통한 CI/CD 가동 테스트 정의
├── data/
│   └── .gitkeep                # DB가 안전하게 영속화될 로컬 마운트 디렉터리
├── src/
│   ├── __init__.py
│   ├── database.py             # SQLite DB 연결 설정, 스키마 구성, 복합 인덱싱, 벌크 UPSERT
│   ├── metrics.py              # 모니터링 메트릭 적재기 및 경고 엔진 정의
│   ├── models.py               # Pydantic을 이용한 도서 데이터 수집 규격 설계
│   ├── queue_manager.py        # Redis 연결 통제 및 중복 차단 관리
│   └── scraper.py              # BeautifulSoup 파서 엔진 및 다중 스레드 워커 구현
├── tests/
│   ├── __init__.py
│   └── test_pipeline.py        # 유닛 테스트 모음
├── Dockerfile                  # 가볍고 격리된 애플리케이션 가상 이미지 정의
├── docker-compose.yml          # Redis 컨테이너와 크롤러의 실시간 연결 오케스트레이션
├── main.py                     # 수집 작업을 개시하는 실행 시작점
└── requirements.txt            # 파이썬 종속 패키지 목록
